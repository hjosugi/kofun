# 言語仕様 v0 — read / edit / take

このリポジトリは**実行可能仕様**です。散文の仕様書ではなく、
「サンプルプログラム + 期待出力」の対で意味論を固定します。

- `langc/langc.py` — 参照実装（oracle）。lex → parse → 所有権検査 → 木構造インタプリタ
- `run_spec.py` — ゴールデンテストランナー（実装非依存）
- `spec/**/*.lang` + `*.expected` — 実行可能仕様 29本

## 使い方

```
python3 run_spec.py                      # 全スペックを参照実装で実行
python3 run_spec.py --filter ownership   # 部分実行
python3 run_spec.py --update             # 期待値を再生成（意図的な仕様変更時のみ）
python3 langc/langc.py run   file.lang   # 検査 + 実行
python3 langc/langc.py check file.lang   # 静的検査のみ
```

**差分テスト**（本命の使い方）。C backend / LLVM backend を作ったら、
同じスペックを別実装に向けて回します。

```
python3 run_spec.py --cmd "./build/mylang run {file}"
```

参照実装と新backendの出力が1バイトでも違えば落ちます。
インタプリタを先に完成させる理由はこれです。仕様のoracleになります。

## 所有権モデル

lifetimeを構文に出さない。参照は第二級（返却・構造体格納は不可）。
プログラマが書くのは**引数規約3つ**だけです。

| 規約 | 意味 | 呼び出し側 | 呼ばれた側 |
|---|---|---|---|
| `read` | 共有借用 | 値は使い続けられる | 読むだけ。変更・move不可 |
| `edit` | 排他借用 | `var` が必要。呼び出し中は他から触れない | 変更可。move不可 |
| `take` | 所有権移動 | 元の変数は以降使えない | 所有する。変更もmoveも可 |

対応関係（この3分法は独立に4系統が到達した収束点です）。

| 本言語 | Hylo | Mojo | Swift | Rust |
|---|---|---|---|---|
| `read` | `let` | `read` | `borrowing` | `&T` |
| `edit` | `inout` | `mut` | `inout` | `&mut T` |
| `take` | `sink` | `owned` | `consuming` | `T` |

### 束縛の種類と権限

| 種類 | 変更 | 再代入 | move |
|---|---|---|---|
| `let` | 不可 | 不可 | 可 |
| `var` | 可 | 可 | 可（再代入で復活） |
| `read` 引数 | 不可 | 不可 | 不可 |
| `edit` 引数 | 可 | 不可 | 不可 |
| `take` 引数 | 可 | 不可 | 可 |
| ループ変数 | 不可 | 不可 | 未決（E007参照） |

### 確定した規則

1. **moveは束縛単位で追跡する。** move後の使用はE001。
2. **`var` は再代入でmove状態から復活する。** 別の値を入れたのだから当然。
3. **分岐後のjoinは保守的。** どちらか一方の枝でmoveされたら、合流後はmove済み。
   `if` に `else` が無い場合も同じ（`spec/05_ownership_errors/048`）。
4. **ループ外で宣言した束縛をループ内でmoveできない**（E005）。2周目が壊れるため。
   これは型情報なしで健全に判定できる数少ない規則なので、v0から入れています。
5. **排他性は呼び出し単位で検査する。** `edit` で渡した変数を同じ呼び出しの
   他の引数でも使うとE003。`merge(xs, xs)` は弾かれます。
6. **`edit` は呼び出し側のセルを共有する。** 実装上はcell渡しで、コピーバックではありません。
7. **クロージャは可変束縛を捕獲できない**（E011）。v0の制限。
   捕獲したエイリアス経由の変更と直接の変更が競合するため。

## エラーカタログ

出力形式は固定です。`error[CODE] line N: message`

| コード | 内容 |
|---|---|
| E001 | move済みの値を使用（moveした行を併記） |
| E002 | `edit` 引数に `let` 束縛を渡した |
| E003 | 排他借用の衝突（同一呼び出し内でのエイリアス） |
| E004 | `read` 引数 / ループ変数を変更しようとした |
| E005 | ループ外の束縛をループ内でmoveした |
| E006 | 借用中の引数をmoveしようとした |
| E007 | **予約**（未決事項を参照） |
| E010 | `let` 束縛への再代入 |
| E011 | クロージャによる可変束縛の捕獲 |
| E020 | 未定義の名前 |
| E021 | 引数の個数が合わない |
| E022 | 未定義のメソッド |

同一の診断は1回だけ出します（重複抑止済み）。

## 未決事項（型検査器で決める）

### E007: 借用したコレクションから要素をmoveできるか

```
fn first_even(read xs) -> Option[Int] {
    for x in xs {
        if x % 2 == 0 { return Some(x) }   // x をmoveしている
    }
    None
}
```

`Int` ならコピーなので健全。`List[String]` なら借用元のエイリアスになるので不健全。
**型がないと判定できません。** v0のoracleは許可し、判断を保留しています。

候補は2つ。

- **案A: Copy/非Copy を型で分ける。** Rust/Swift/Mojo/Hylo が全部これ。実績はあるが概念が1つ増える。
- **案B: 反復対象が名前なら借用、一時値なら所有。** `for x in xs` は借用、
  `for x in xs.clone()` は所有。sigilは不要だが、離れた場所の情報で意味が変わる。
  「直感的でひねらない」という目標には反する。

**現時点の判断: 案A。** ただし `Copy` をユーザーが実装できる trait にはせず、
組み込みスカラ（Int / Float / Bool / Unit）に固定する。ユーザー型は常に非Copy。
概念を1つ増やす代償を、ユーザーが意識する場面をゼロにすることで払います。

### その他

- **matchの網羅性検査が未実装。** v0は実行時エラー（`spec/06_runtime/052`）。
  型検査器が入った時点で静的エラーに格上げします。
- **型検査そのものが未実装。** v0は動的（`spec/06_runtime/053`）。
  HM推論 + 引数規約は独立に動くので、所有権規則を先に固めるこの順序は正しい。
- **`edit` の再借用（reborrow）を明示的に扱っていない。**
  `edit` 引数を別関数の `edit` 引数へ渡すのは通っています（`forward(edit xs) { inner(xs) }`）。
  排他性が推移的に保たれるので健全ですが、規則として明文化が必要。

## 構文で確定したこと

- **改行が文の区切り。** ただし `(` `[` の内側では無意味。`{` の内側では有意。
- **先頭演算子による行継続。** 次の行が二項演算子で始まるなら継続扱い。
  多行パイプラインが書けます。`-` だけは除外（単項マイナスで始まる文と曖昧なため）。
- **`|>` はパーサーで書き換える。** `x |> f()` は `f(x)`、`x |> f(a)` は `f(x, a)`。
  実行時のコストはゼロ。所有権上は通常の呼び出しと同じなので、
  `read` 引数のパイプラインは値を消費しません。
- **`?` はResultとOptionに対して動く。** `Err` / `None` は関数境界まで脱出。
- **コンストラクタは大文字始まり。** パターン内で束縛と区別するための規約。
- **`...` は展開（リテラル内）と残余（パターン内）の両方。**

## 設計上の非目標（v0で意図的に入れないもの）

- reactive（signal / computed / effect）は言語コアに入れない。ライブラリ層へ。
- actorも同様。structured concurrencyの上に載せる。
- algebraic effectsはv2候補。v1は `?` と `use` 相当の糖衣1個で出す。
- HKTベースの汎用monadは入れない。「直感的」と両立しないため。
- セルフホストは封印。
